📄 whatsapp_adapter.py — Abstração do provedor

class WhatsAppProvider(ABC):
    """Interface comum — Evolution API ou Z-API."""

    @abstractmethod
    async def send_text(self, phone: str, msg: str) → dict: ...

    @abstractmethod
    async def send_template(self, phone: str, template: str, vars: dict) → dict: ...

    @abstractmethod
    async def send_file(self, phone: str, url: str, caption: str) → dict: ...


class EvolutionAPIProvider(WhatsAppProvider):
    """Evolution API self-hosted."""
    base_url = settings.EVOLUTION_API_URL   # ex: http://localhost:8080
    api_key  = settings.EVOLUTION_API_KEY
    instance = settings.EVOLUTION_INSTANCE

    async def send_text(self, phone, msg):
        return await self._post(
            f"/message/sendText/{self.instance}",
            {"number": phone, "text": msg}
        )


class ZAPIProvider(WhatsAppProvider):
    """Z-API SaaS."""
    base_url  = "https://api.z-api.io/instances"
    instance  = settings.ZAPI_INSTANCE_ID
    token     = settings.ZAPI_TOKEN

    async def send_text(self, phone, msg):
        return await self._post(
            f"/token/{self.token}/send-text",
            {"phone": phone, "message": msg}
        )


def get_whatsapp_provider() → WhatsAppProvider:
    """Factory — lê configuração do banco (editável no frontend)."""
    config = db.get_setting("whatsapp_provider")
    return EvolutionAPIProvider() if config == "evolution" else ZAPIProvider()

Todas as automações rodam em Python puro via Celery + Redis (tarefas assíncronas) e APScheduler (agendamentos recorrentes). Nenhum Make, Zapier ou n8n necessário. Regras editáveis pelo gestor no painel web.

Fluxo 1 — Lead novo via formulário ou WhatsApp

⚡ Trigger: novo lead capturado

FORM / WA recebido → Cria Lead no PostgreSQL → Calcula lead score → Score ≥ 60? → WA automático <2min → Notifica vendedor → Agenda visita no Google Cal
Fluxo 2 — Régua de lembretes pré-evento

🗓 Trigger: Celery Beat verifica diariamente às 08:00

Celery Beat 08:00 → Query eventos próximos → D-90: checklist planejamento → D-30: confirmação buffet/fornec. → D-7: instruções acesso → D-1: checklist final + horários → WA via Evolution/Z-API
Fluxo 3 — Cobrança de parcelas

💰 Trigger: Asaas webhook + Celery schedule

Parcela gerada no Asaas → D-5: WA "sua parcela vence em 5 dias" → D-1: WA + link boleto/Pix → D+0 vencimento: alerta final → Pago? (webhook Asaas) → Confirma pagamento no CRM
Fluxo 4 — Pós-evento e reputação

⭐ Trigger: evento marcado como "realizado"

Status → Evento Realizado → D+1: WA agradecimento → D+3: pesquisa satisfação (NPS) → NPS ≥ 8? → Link Google Review → M+11: "Novo aniversário?" → Re-entrada no funil
Fluxo 5 — Fornecedores

🤝 Trigger: evento confirmado

Contrato assinado → D-30: e-mail confirmação fornecedor → D-7: WA horário chegada + acesso → D-1: lembrete final → Checklist interno equipe

Ação por faixa de score

🔴 Score < 30 — Frio

Entra em e-mail marketing geral. WA apenas com novidades e datas disponíveis. Cadência mensal.

🟡 Score 30–59 — Morno

Sequência de nutrição automática. WA a cada 15 dias. Convite para tour virtual 360°.

🟢 Score ≥ 60 — Quente

Follow-up pessoal imediato. WA em <5 min. Proposta personalizada + convite para visita presencial.
